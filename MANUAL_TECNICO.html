4. Script de Generación de la Base de Datos

-- RAVE Platform - Script SQL (equivalente conceptual)
-- La aplicación usa localStorage como base de datos NoSQL

CREATE TABLE users (
  id VARCHAR(20) PRIMARY KEY,
  email VARCHAR(255) UNIQUE NOT NULL,
  password VARCHAR(255) NOT NULL,
  displayName VARCHAR(100) NOT NULL,
  role ENUM('user', 'organizer') DEFAULT 'user',
  createdAt DATETIME DEFAULT CURRENT_TIMESTAMP,
  brand_name VARCHAR(100),
  brand_bio TEXT,
  brand_logo VARCHAR(500),
  brand_cover VARCHAR(500),
  brand_city VARCHAR(100),
  brand_instagram VARCHAR(100),
  brand_website VARCHAR(200),
  brand_founded VARCHAR(4)
);

CREATE TABLE events (
  id VARCHAR(20) PRIMARY KEY,
  title VARCHAR(60) NOT NULL,
  description TEXT,
  date DATE NOT NULL,
  time TIME NOT NULL,
  duration INT DEFAULT 6,
  location VARCHAR(200) NOT NULL,
  address VARCHAR(200),
  city VARCHAR(100),
  price DECIMAL(10,2) DEFAULT 0,
  capacity INT DEFAULT 500 CHECK(capacity >= 10),
  genre VARCHAR(50),
  imageUrl VARCHAR(500),
  imagePos INT DEFAULT 50,
  minAge INT DEFAULT 18,
  organizerId VARCHAR(20) NOT NULL,
  status VARCHAR(20) DEFAULT 'active',
  ticketsSold INT DEFAULT 0,
  createdAt DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (organizerId) REFERENCES users(id)
);

CREATE TABLE tickets (
  id VARCHAR(20) PRIMARY KEY,
  eventId VARCHAR(20) NOT NULL,
  userId VARCHAR(20) NOT NULL,
  purchaseDate DATETIME DEFAULT CURRENT_TIMESTAMP,
  status ENUM('valid', 'used') DEFAULT 'valid',
  qrCode VARCHAR(50) UNIQUE NOT NULL,
  usedAt DATETIME,
  FOREIGN KEY (eventId) REFERENCES events(id),
  FOREIGN KEY (userId) REFERENCES users(id),
  UNIQUE(eventId, userId)
);

CREATE TABLE notifications (
  id VARCHAR(20) PRIMARY KEY,
  userId VARCHAR(20) NOT NULL,
  type VARCHAR(50),
  title VARCHAR(200) NOT NULL,
  message TEXT,
  eventId VARCHAR(20),
  image VARCHAR(500),
  isRead BOOLEAN DEFAULT FALSE,
  createdAt DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (userId) REFERENCES users(id)
);

CREATE TABLE subscriptions (
  id VARCHAR(20) PRIMARY KEY,
  userId VARCHAR(20) NOT NULL,
  eventId VARCHAR(20) NOT NULL,
  createdAt DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (userId) REFERENCES users(id),
  UNIQUE(userId, eventId)
);

b. Casos de Uso Extendidos

CU04: Comprar Ticket

CampoDescripción
NombreComprar Ticket
ActorRaver
PrecondiciónEl usuario está autenticado con rol 'user' y está en la página de detalle de un evento
Flujo principal1. El Raver ve el detalle del evento con precio y disponibilidad.
2. Hace clic en "Comprar Ticket".
3. El sistema verifica que no tenga ya un ticket para ese evento.
4. Crea un ticket con código QR único (formato RAVE-XXXXXXXX).
5. Incrementa el contador de tickets vendidos.
6. Genera una notificación de compra.
7. Muestra confirmación "¡Ticket comprado!" con enlace a Mis Tickets.
Flujo alternativo 1Si no está autenticado → redirige a /login
Flujo alternativo 2Si ya tiene ticket → muestra "Ya tienes ticket" con enlace a Mis Tickets
Flujo alternativo 3Si el evento está agotado → botón "Agotado" deshabilitado
Flujo alternativo 4Si es el organizador del evento → botón deshabilitado "No puedes comprar tu propio evento"
PostcondiciónSe crea registro en Tickets con status='valid' y qrCode único

CU11: Crear Evento

CampoDescripción
NombreCrear Evento
ActorOrganizador
PrecondiciónEl usuario está autenticado con rol 'organizer'
Flujo principal1. Accede a "Crear Evento".
2. Paso 1: Ingresa nombre (máx 60 chars), selecciona género de 8 opciones, escribe descripción (mín 20 chars), opcionalmente agrega line-up.
3. Paso 2: Selecciona fecha (mínimo hoy), hora inicio, duración (3h/6h/10h/12h/24h/48h o personalizada), venue, ciudad.
4. Paso 3: Define precio, capacidad (10-50,000), edad mínima (16/18/21).
5. Paso 4: Sube imagen o selecciona de galería, ajusta posición vertical.
6. Hace clic en "Publicar Evento".
7. El sistema crea el evento y redirige al Dashboard.
Flujo alternativoSi la validación falla en cualquier paso → muestra errores en rojo, no permite avanzar
PostcondiciónSe crea registro en Events con status='active' y ticketsSold=0

CU14: Escanear QR

CampoDescripción
NombreValidar Entrada con QR
ActorOrganizador
PrecondiciónEl Organizador tiene al menos un evento creado
Flujo principal1. Accede al Scanner desde Dashboard o Mis Eventos.
2. Ve nombre del evento y stats en vivo (check-ins / total).
3. Selecciona modo Cámara o Manual.
4. Ingresa el código QR del ticket.
5. El sistema busca el ticket por qrCode y eventId.
6. Si válido → muestra "¡Acceso Concedido!" en verde, marca ticket como 'used'.
7. Contador de check-ins se actualiza.
Flujo alternativo 1Si código no existe → "Ticket inválido" en rojo
Flujo alternativo 2Si ticket ya usado → "Ticket ya utilizado" en rojo
PostcondiciónTicket cambia status='valid' → status='used', se registra usedAt
